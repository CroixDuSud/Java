Les sections suivantes décrivent la manière dont le pilote JDBC de Microsoft SQL Server 2005 prend en charge les fonctionnalités d'API optionnelles de J2EE et JDBC 3.0. Les exemples de code source contenus dans ce système d'aide constituent une bonne référence pour la prise en main de ces fonctionnalités.

Commencez par vous assurer que votre environnement Java (JDK, JRE) inclut le package javax.sql. Il s'agit d'un package obligatoire pour toute application JDBC utilisant l'API en option. JDK 1.4 et ses versions supérieures intègrent déjà ce package, de sorte que vous ne devez pas l'installer séparément.

Nom du pilote

Le nom de classe du pilote est com.microsoft.sqlserver.jdbc.SQLServerDriver. Le pilote se trouve dans le fichier sqljdbc.jar.

Le nom de classe est utilisé à chaque chargement du pilote à l'aide de la classe JDBC DriverManager. Il est également utilisé chaque fois que vous devez spécifier le nom de classe du pilote dans une configuration de pilote. Par exemple, la configuration d'une source de données dans un serveur d'applications J2EE peut nécessiter la saisie du nom de classe du pilote.

Sources de données

Le pilote JDBC prend en charge les sources de données J2EE/JDBC 3.0. La classe SQLServerXADataSource du pilote JDBC est implémentée par com.microsoft.sqlserver.jdbc.SQLServerXADataSource.

Noms de sources de données

Vous pouvez établir des connexions de base de données à l'aide de sources de données. Le tableau suivant décrit les sources de données disponibles avec le pilote JDBC :

Type de source de données Nom de la classe Description

DataSource

com.microsoft.sqlserver.jdbc.SQLServerDataSource

Source de données non regroupables.

ConnectionPoolDataSource

com.microsoft.sqlserver.jdbc.SQLServerConnectionPoolDataSource

Source de données pour la configuration des regroupements de connexions du serveur d'application J2EE. Généralement utilisée lorsque l'application s'exécute dans un serveur d'applications J2EE.

XADataSource

com.microsoft.sqlserver.jdbc.SQLServerXADataSource

Source de données pour la configuration de sources de données J2EE XA. Généralement utilisée lorsque l'application s'exécute dans un serveur d'applications J2EE et un gestionnaire de transactions XA.

Propriétés de la source de données

Toutes les sources de données offrent la possibilité de définir et d'obtenir une propriété associée au jeu de propriétés du pilote sous-jacent.

Exemples :

setServerName("localhost");

setDatabaseName("AdventureWorks");

La section suivante montre comment une application se connecte à l'aide d'une source de données :

initialize JNDI ..
Context ctx = new InitialContext(System.getProperties());
...
DataSource ds = (DataSource) ctx.lookup("MyDataSource");
Connection c = ds.getConnection("user", "pwd");

Pour plus d'informations sur les propriétés des sources de données, voir Définition des propriétés de la source de données.

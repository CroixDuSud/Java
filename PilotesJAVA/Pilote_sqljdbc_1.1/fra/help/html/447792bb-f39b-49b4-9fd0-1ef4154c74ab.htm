Pour accéder aux données d'une base de données SQL Server à l'aide du pilote JDBC de Microsoft de SQL Server 2005, vous devez installer les composants suivants sur votre ordinateur :

  • Microsoft Pilote JDBC de SQL Server 2005

  • Java Runtime Environment

Configuration requise pour Java Runtime Environment

Les fonctionnalités de JDBC 3.0 requièrent la version 1.4 ou une version ultérieure de Java Runtime Environment (JRE). Par conséquent, le pilote JDBC requiert également la version 1.4 ou une version ultérieure de JRE. Le pilote JDBC, conçu pour fonctionner sur toutes les principales machines virtuelles, est officiellement pris en charge par Sun JRE 1.4 ou ultérieur.

Configuration requise pour SQL Server

Le pilote JDBC, spécifiquement conçu pour utiliser les fonctionnalités de SQL Server 2005, offre également une compatibilité descendante avec SQL Server 2000, notamment la version 64 bits.

Configuration requise pour le système d'exploitation

Le pilote JDBC est conçu pour fonctionner sur tout système d'exploitation prenant en charge l'utilisation d'une machine virtuelle Java (Java Virtual Machine, JVM). Toutefois, seuls les systèmes d'exploitation Sun Solaris, Red Hat Linux et Windows 2000 ou ultérieur sont officiellement pris en charge.

Langages pris en charge

Le pilote JDBC prend en charge tous les classements de colonnes SQL Server.

Pour plus d'informations sur les classements, voir la rubrique « Classements » dans la documentation en ligne de SQL Server.

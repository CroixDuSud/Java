Cette version 1.1 du pilote JDBC de Microsoft de SQL Server 2005 inclut plusieurs nouvelles fonctionnalités et améliorations. Les sections suivantes décrivent les améliorations apportées à cette version du pilote JDBC.

Authentification intégrée

Désormais, le pilote JDBC prend en charge l'utilisation de l'authentification intégrée de type 2 sur les systèmes d'exploitation Microsoft Windows. Pour obtenir des informations sur la configuration d'un ordinateur pour utiliser l'authentification intégrée, voir Création de l'URL de connexion. Pour obtenir des informations sur l'utilisation de la propriété de chaîne de connexion integratedSecurity dans une application Java, voir Définition des propriétés de connexion.

Mise en miroir de bases de données

Désormais, le pilote JDBC prend en charge l'utilisation de la mise en miroir de bases de données SQL Server 2005, fonctionnalité qui permet d'augmenter la disponibilité de la base de données et la redondance des données. Pour plus d'informations sur l'utilisation de la mise en miroir de bases de données avec le pilote JDBC, voir Utilisation de la mise en miroir de bases de données (JDBC).

Propriété de la taille du paquet

Désormais, le pilote JDBC prend en charge l'utilisation de la propriété de chaîne de connexion packetSize, qui permet de définir ou d'extraire la taille de paquet du réseau utilisé pour communiquer avec SQL Server. Pour plus d'informations sur la nouvelle propriété packetSize, consultez Définition des propriétés de connexion.

Exemples d'applications

Le système d'aide du pilote JDBC inclut désormais une série d'exemples d'applications qui présentent plusieurs techniques d'utilisation du pilote JDBC dans des applications Java. Pour plus d'informations sur les exemples d'applications, voir Exemples d'applications du pilote JDBC.

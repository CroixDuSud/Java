L'une des principales choses à faire avec le pilote JDBC de Microsoft de SQL Server 2005 est d'établir une connexion avec une base de données SQL Server. Toute interaction avec la base de données se produit via l'objet SQLServerConnection et, comme le pilote JDBC présente une architecture très plate, pratiquement tout comportement intéressant touche l'objet SQLServerConnection.

Les rubriques de cette section décrivent la création et le fonctionnement d'une connexion à une base de données SQL Server.

Dans cette section

Rubrique Description

Création de l'URL de connexion

Décrit la manière de créer une URL pour se connecter à une base de données SQL Server. Décrit également la connexion à des instances nommées d'une base de données SQL Server.

Définition des propriétés de connexion

Décrit les diverses propriétés de connexion et la façon de les utiliser pour se connecter à une base de données SQL Server.

Définition des propriétés de la source de données

Décrit la manière d'utiliser des sources de données dans un environnement J2EE.

Travail sur une connexion

Décrit les diverses méthodes permettant de créer une instance d'une connexion à une base de données SQL Server.

Utilisation d'un regroupement de connexions

Décrit la manière dont le pilote JDBC prend en charge l'utilisation d'un regroupement de connexions.

Utilisation de la mise en miroir de bases de données (JDBC)

Décrit la manière dont le pilote JDBC prend en charge l'utilisation d'une mise en miroir de bases de données.

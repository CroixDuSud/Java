Lorsque vous travaillez sur des objets du pilote JDBC de Microsoft SQL Server 2005, notamment SQLServerResultSet ou l'un des objets Statement, tels que SQLServerStatement, SQLServerPreparedStatement ou SQLServerCallableStatement, vous devez explicitement les fermer en utilisant leurs méthodes close lorsqu'ils ne sont plus nécessaires. Cela améliore les performances en libérant des ressources du pilote et du serveur dès que possible, sans attendre que le garbage collector de la machine virtuelle Java le fasse pour vous.

La fermeture des objets est particulièrement importante pour le maintien d'un bon accès simultané sur le serveur lorsque vous utilisez des verrous de défilement. Des verrous de défilement dans le tampon d'extraction auquel vous avez accédé en dernier sont maintenus jusqu'à la fermeture du jeu de résultats. De même, les descripteurs d'instruction préparés sont maintenus jusqu'à la fermeture de l'instruction. Si vous réutilisez une connexion pour plusieurs instructions, la fermeture des instructions avant de les laisser dépasser l'étendue permet au serveur de nettoyer les descripteurs préparés précédemment.

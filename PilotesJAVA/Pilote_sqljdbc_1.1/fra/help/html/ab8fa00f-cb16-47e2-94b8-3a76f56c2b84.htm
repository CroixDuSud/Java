Il existe de nombreuses différences entre les types de données du langage de programmation Java et de SQL Server 2005. Le pilote JDBC de Microsoft de SQL Server 2005 vous aide à maîtriser les différences entre ces différents types de conversions.

Types de caractères

Les types de données chaînes de caractères JDBC sont CHAR, VARCHAR et LONGVARCHAR.

Type Description

Fixed-length

Les types de données SQL Server char et nchar correspondent directement au type JDBC CHAR. Il s'agit de types de longueur fixe avec une marge fournie par le serveur au cas où la colonne a SET ANSI_PADDING ON. La marge est toujours activée pour nchar, mais pour char, si les colonnes char du serveur ne sont pas complétées, le pilote JDBC ajoute la marge.

Variable-length

Les types SQL Server varchar et nvarchar correspondent directement au type JDBC VARCHAR.

Long

Les types SQL Server text et ntext correspondent directement au type JDBC LONGVARCHAR. Ces types sont désapprouvés pour SQL Server 2005, vous devez donc utiliser des types de grande valeur à la place.

L'utilisation des méthodes update<Numeric Type> et updateObject (int, java.lang.Object) est impossible sur les colonnes text et ntext du serveur. Cependant, l'utilisation de la méthode setObject avec un type de conversion de caractère spécifié est prise en charge sur les colonnes text et ntext du serveur.

Types de chaînes binaires

Les types de chaînes binaires JDBC sont BINARY, VARBINARY et LONGVARBINARY.

Type Description

Fixed-length

Le type SQL Server binary correspond directement au type JDBC BINARY. Il s'agit d'un type de longueur fixe avec une marge fournie par le serveur au cas où la colonne a SET ANSI_PADDING ON. Si les colonnes char du serveur ne disposent pas de marge, le pilote JDBC ajoute la marge.

Le type SQL Server timestamp est un type JDBC BINARY avec une longueur fixe de 8 octets.

Variable-length

Le type SQL Server varbinary correspond au type JDBC VARBINARY.

Long

Le type SQL Server image correspond au type JDBC LONGVARBINARY. Ce type est désapprouvé pour SQL Server 2005, vous devez donc utiliser un type de grande valeur à la place.

Le type udt dans SQL Server 2005 correspond à un type JDBC LONGBINARY.

Types numériques exacts

Les types numériques exacts JDBC correspondent directement aux types SQL Server correspondants.

Type Description

BIT

Le type JDBC BIT représente un bit unique qui peut être 0 ou 1. Cela correspond à un type SQL Server bit.

TINYINT

Le type JDBC TINYINT représente un octet unique. Cela correspond à un type SQL Server tinyint.

SMALLINT

Le type JDBC SMALLINT représente un entier signé de 16 octets. Cela correspond à un type SQL Server smallint.

INTEGER

Le type JDBC INTEGER représente un entier signé de 32 octets. Cela correspond à un type SQL Server integer.

BIGINT

Le type JDBC BIGINT représente un entier signé de 64 octets. Cela correspond à un type SQL Server bigint.

NUMERIC

Le type JDBC NUMERIC représente une valeur décimale à précision fixe qui conserve les valeurs de précision identique. Le type NUMERIC correspond au type SQL Server numeric.

DECIMAL

Le type JDBC DECIMAL représente une valeur décimale à précision fixe qui conserve au minimum les valeurs de la précision spécifiée. Le type DECIMAL correspond au type SQL Server decimal.

Le type JDBC DECIMAL correspond également aux types SQL Server money et smallmoney, qui sont des types decimal à précision fixe stockés respectivement sur 8 et 4 octets.

Types numériques approximatifs

Les types numériques approximatifs JDBC sont REAL, DOUBLE et FLOAT.

Type Description

REAL

Le type JDBC REAL a sept chiffres de précision (précision simple) et correspond directement au type SQL Server real.

DOUBLE

Le type JDBC DOUBLE a 15 chiffres de précision (précision double) et correspond au type SQL Server float. Le type JDBC FLOAT est un synonyme de DOUBLE. La confusion entre FLOAT et DOUBLE étant possible, DOUBLE est recommandé.

Types de date/d'heure

Le type JDBC TIMESTAMP correspond aux types SQL Server datetime et smalldatetime. Le type datetime est stocké dans deux entiers de 4 octets. Le type smalldatetime conserve les mêmes informations (date et heure) mais avec une précision inférieure, dans deux petits entiers de 2 octets.

Remarque : Le type SQL Server timestamp est un type de chaîne binaire à longueur fixe. Il ne correspond à aucun type JDBC time : DATE, TIME ou TIMESTAMP.

Mappage de type personnalisé

La fonction de mappage de type personnalisé de JDBC qui utilise les interfaces SQLData pour les types avancés JDBC (UDT, Struct, etc.) n'est pas implémentée dans le pilote JDBC.

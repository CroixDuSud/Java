Le pilote JDBC de Microsoft SQL Server 2005 utilise des verrous de ligne SQL Server. Ces derniers implémentent un contrôle d'accès simultané entre plusieurs utilisateurs modifiant simultanément une base de données. Par défaut, les transactions et les verrous sont gérés individuellement pour chaque connexion. Par exemple, si une application ouvre deux connexions JDBC, les verrous acquis par une connexion ne peuvent pas être partagés avec l'autre connexion. Aucune connexion ne peut acquérir de verrous susceptibles d'être en conflit avec des verrous placés par l'autre connexion.

Remarque : En cas d'utilisation du verrouillage de ligne, toutes les lignes du tampon d'extraction sont verrouillées, de sorte qu'un paramétrage trop étendu de la taille d'extraction peut affecter l'accès simultané.

Le verrouillage permet d'assurer l'intégrité des transactions et la cohérence de la base de données. Le verrouillage empêche les utilisateurs de lire des données en cours de modification par d'autres utilisateurs et empêche plusieurs utilisateurs de modifier les mêmes données en même temps. En cas de non-utilisation du verrouillage, les données de la base de données risquent de devenir logiquement incorrectes et les requêtes appliquées aux données, de produire des résultats inattendus.

Remarque : Pour plus d'informations sur le verrouillage de données dans SQL Server, voir « Verrouillage dans Moteur de base de données » dans la documentation en ligne de SQL Server.

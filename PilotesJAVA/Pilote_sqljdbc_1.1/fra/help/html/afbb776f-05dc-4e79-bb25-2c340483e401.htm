Le traitement de transactions est une condition obligatoire pour toutes les applications qui doivent garantir la cohérence de leurs données permanentes. Avec le pilote JDBC de Microsoft de SQL Server 2005, le traitement de transactions peut être effectué localement ou de manière distribuée. Les transactions sont des modules d'exécution ACID (Atomicité, Cohérence, Isolation et Durabilité).

Les rubriques de cette section décrivent la manière dont le pilote JDBC prend en charge les transactions, y compris les niveaux d'isolation, les points d'enregistrement de transactions et la fonction holdability du jeu de résultats.
